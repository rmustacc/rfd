DDEESSCCRRIIPPTTIIOONN
     The ddddii__uuffmm__iimmaaggee__sseett__ddeesscc() and ddddii__uuffmm__iimmaaggee__sseett__mmiisscc() functions are
     used by device drivers to set information about a firmare image on the
     image structure _u_i_p as a part of implementing their
     ddi_ufm_op_fill_image(9E) entry point. For more information on images and
     the use of these functions, see the descrption of the
     ddddii__uuffmm__oopp__ffiillll__iimmaaggee() function in ddi_ufm(9E).

     The ddddii__uuffmm__iimmaaggee__sseett__ddeesscc() function sets the description of the
     fimrware image. This description is intended for administrators and
     should convey the intended use of the image.

     The ddddii__uuffmm__iimmaaggee__sseett__mmiisscc() function is used by drivers to set ancillary
     key-value data that may be useful to a consumer. The driver should create
     an nvlist for this purpose with nvlist_alloc(9F.) Once the driver passes
     the nvlist to the ddddii__uuffmm__iimmaaggee__sseett__mmiisscc() function, then the driver must
     not manipulate or free the nvlist at all.  It is the property of the UFM
     subsystem.
